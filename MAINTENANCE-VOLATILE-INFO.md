# 陳腐化しやすい情報（AI事情）の扱いと更新手順

本書は「論理的思考とAI協働の考え方」を主対象とし、特定ベンダー/特定モデルの仕様表や価格表を提供することを目的にしません。

一方で、読者の判断に影響しやすい「陳腐化しやすい情報」（モデル名・料金・ツール一覧・規制の施行状況など）が本文に混入すると、時間経過で誤解を招く可能性があります。この文書は、当該情報の取り扱い方針と更新手順を定義します（Issue #69）。

## 1. 方針（本文の書き方）

### 原則
- 本文は「評価軸」「判断基準」「プロセス」を中心に記述する。
- 固有名詞・数値・年次断定は、必要最小限に抑える。

### 例示が必要な場合のルール
固有名詞/数値を本文に残す必要がある場合は、次を必須とします。

- **執筆時点/更新日**を明記する（例: `執筆時点: 2026-01`）
- **参照先（一次情報）**を明記する（公式サイト/公式ドキュメント等）
- **適用範囲**を明記する（例: 「プランにより異なる」「契約形態により異なる」）

## 2. 「陳腐化しやすい情報」の定義

特に注意する対象は以下です。

- **モデル名・プラン名・機能名**（例: モデル世代、提供形態の呼称）
- **価格・課金体系・コンテキスト長等の数値**
- **ツール一覧**（情報検証、AI検出、エージェント関連など）
- **年次断定**（例: 「2024年版」「2025年の〜」「◯年に施行」）
- **規制・制度の状況**（施行日/適用範囲/ガイドラインの最新版など）

## 3. 更新運用（決め）

### 更新頻度
- 原則: **半年に1回**（棚卸し → 必要な修正をPR化）
- 例外: 本文に新たな固有名詞/数値を追加した場合は、そのPR内で「執筆時点/参照先」を付与し、棚卸し対象に追加する

### 担当
- 本リポジトリの**メンテナ（レビュー担当）**が実施する

### 主な更新対象ファイル
- `src/chapters/`（本文の原稿）
- `docs/chapters/`（公開ページ）
- `src/practical-guides/` など、具体ツールに触れる補助資料（存在する場合）

## 4. 棚卸し手順（検索キー例）

以下の例で横断検索し、該当箇所を抽出します（必要に応じて追加）。

```bash
# 年次断定・版の表現
rg -n \"(20[0-9]{2}年版|20[0-9]{2}[-〜]20[0-9]{2}|20[0-9]{2}年.*(施行|発表))\" docs src

# モデル/ツールの固有名詞（例）
rg -n \"(GPT|Claude|Gemini|LLaMA|Copilot|AI検出|検出ツール|料金|tokens|コンテキスト)\" docs src
```

## 5. 更新後の確認（docs/ 公開ページ）

`docs/` 配下の編集を含む場合は、最低限以下を実行します。

```bash
npm install
npm run lint:light
npm run check-links
```

必要に応じてビルドも確認します。

```bash
npm run build
```

## 6. 参照先（一次情報の例）

リンク先は変更される可能性があるため、到達できない場合は差し替えます。

- OpenAI: 価格/プラン（要確認） `https://openai.com/pricing`
- Anthropic: 価格/プラン `https://claude.com/pricing`
- Google: Gemini API 価格/プラン `https://ai.google.dev/gemini-api/docs/pricing`
- Google Cloud: Vertex AI 価格/プラン `https://cloud.google.com/vertex-ai/pricing`
- EU: AIに関する規制枠組み（欧州委員会） `https://digital-strategy.ec.europa.eu/en/policies/regulatory-framework-ai`
