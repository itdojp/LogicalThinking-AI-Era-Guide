# 1. 書籍の概要

- 想定読者（docs/index.md ベース）
  - 生成AIツールを業務で使い始めた、または使う予定の一般会社員（営業・企画・人事・経理・マーケティング等）。
  - 論理的思考に苦手意識がある読者。
  - 年齢層は 20代後半〜40代前半の中堅社員を想定。
- 本書で達成しようとしているゴール（docs/index.md ベース）
  - 「結論→理由→具体例」のような論理構造を意識して説明できるようになる。
  - 生成AIに対して目的・前提・制約条件を明確にした指示を出し、アウトプットを評価・修正しながら活用できるようになる。
  - 前提・根拠・反論可能性といった観点で主張を整理し、論理に基づく議論ができるようになる。
  - 選択肢と評価軸を可視化し、納得感のある意思決定プロセスを説明できるようになる。

## 3. 構成上の気になる点（候補）

- 目次（docs/index.md）・サイドバー（docs/_data/navigation.yml）・各章タイトル（front matter）・章内見出し番号（例: `7.1`）の整合が崩れている章がある。
  - 例: 第7章の途中見出しが `12.3` 以降になっている。第8章の見出しが `7.1` から始まっている。第12章の見出しが `11.1` から始まっている。
  - 例: `chapter09` の章タイトルが `第8章` になっており、`第8章` が重複して見える。
- 第9章以降（`chapter09`〜`chapter14`）は、章タイトルと章番号の対応関係が読者視点で追いにくい。
  - 章番号が連番になっていない（または欠けて見える）ため、参照・会話・復習が難しくなる。
- 第3部（ビジネス現場での実践）の章立てについて、目次記載（文書作成／プレゼン／交渉／会議／意思決定／日常業務）と実ファイルの内容が一致しているか要確認の箇所がある。
- 章内に「章末演習」「理解度チェック」「次章への橋渡し」が含まれる設計は学習導線として有効だが、全章で同程度に揃っているかは要確認。

## 4. 構成改善の提案（案）

- まず「章番号の正」と「章の並びの正」を決め、`docs/index.md` / `docs/_data/navigation.yml` / 各章 front matter / 章内見出し番号を一括で整合させる。
  - `chapter09`（AIリスク管理）を正式な章として扱うか、別章へ統合するかは要確認。
  - 決定後に、章内見出し番号（例: `7.x` / `8.x` / `11.x` 等）の機械的な修正を行う。
- 第3部の章立て（文書作成／プレゼン／交渉／会議／意思決定／日常業務）について、実ファイルの対応表を作り、欠けている章（「論理的な意思決定」など）がある場合は追加・統合方針を決める。
- 章の学習目標と「主なトピック」の対応を、章ごとに 1 対 1 で確認できるよう、章冒頭に「この章で扱う範囲（含む/含まない）」を 2〜4 行で追記する案を検討する（追記は Stage2 以降の編集スコープ）。
