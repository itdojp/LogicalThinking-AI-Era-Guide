# 付録A：実践的なツール・テンプレート集

## A.1 論理的思考のフレームワーク一覧

### A.1.1 基本的な論理構造

**PREP法テンプレート**
```text
【Point（結論）】
- 最初に結論を明確に述べる

【Reason（理由）】  
- なぜその結論に至ったのかの理由

【Example（具体例）】
- 具体的な事例・データ・経験

【Point（結論の再確認）】
- 結論を再度確認・強調
```

**三段論法チェックリスト**
```text
【大前提】
□ 一般的に受け入れられる原則・法則
□ 論理的に妥当な前提

【小前提】
□ 具体的な事実・状況
□ 大前提に当てはまる条件

【結論】
□ 大前提と小前提から論理的に導かれる
□ 飛躍や矛盾がない
```

### A.1.2 問題解決フレームワーク

**5W1H分析テンプレート**
```text
【What】何を：
- 問題・課題の内容
- 達成すべき目標

【Why】なぜ：
- 問題が発生する原因
- 解決する必要性

【Who】誰が：
- 関係者・責任者
- 影響を受ける人

【When】いつ：
- 期限・タイミング
- 時期的制約

【Where】どこで：
- 場所・範囲
- 対象領域

【How】どのように：
- 解決方法・手段
- 実行プロセス
```

**MECE分析チェックリスト**
```text
【Mutually Exclusive（相互排他）】
□ 重複する要素がない
□ 分類が明確に区別される
□ 境界線が明確

【Collectively Exhaustive（完全網羅）】
□ 漏れがない
□ 全体をカバーしている
□ 重要な要素を見落としていない
```

### A.2.2 高度なプロンプト技術

**段階的検討（ステップ思考）テンプレート**

> ここでは「AIの内部思考を開示させる」ことを目的にしません。実務で必要な「結論＋根拠＋検証手順」を明確にするためのテンプレートとして使います。
```text
【課題】[解決したい問題]

【前提】[与えられている事実/制約]（不明は「要確認」とする）

【検討ステップ】
1) 前提の確認（要確認事項の洗い出し）
2) 選択肢の列挙（最低3案）
3) 評価（目的適合/リスク/実行可能性）

【結論】[推奨案]
【根拠】[結論を支える事実・理由]
【検証手順】[一次情報での確認、追加調査、関係者確認など]
```

**Role Playing（役割演技）テンプレート**
```text
【シナリオ設定】
あなたは[役割・職種]として、[状況・背景]の場面にいます。

【キャラクター設定】
- 経験：[年数・専門性]
- 立場：[組織での位置・責任]
- 性格：[判断基準・価値観]

【課題】
[具体的な問題・依頼]

【制約】
- [時間的制約]
- [リソース制約]
- [その他の制約]

この役割として、どのように対応しますか？
理由も含めて説明してください。
```

### A.2.3 プロンプト資産化（台帳）テンプレート

組織でAI活用を継続するためには、よく使うプロンプトや成果物を「再利用できる形」で残すことが重要です。次のテンプレートを基に、用途別に台帳化します。

**プロンプト台帳テンプレート**
```text
【プロンプト名】[一意に識別できる名前]
【目的】[何のために使うか]
【想定利用シーン】[どの業務で、誰が使うか]
【機密区分】[外部AI可/不可、抽象化の要否]

【入力】[与える材料（例: 事実メモ、制約、前提）]
【制約】[やってはいけないこと、断定禁止、根拠必須など]
【出力形式】[見出し、表、箇条書き、文字数、ファイル形式など]
【受け入れ基準】[CARE + 章固有KPI など]

【検証結果】[成功例/失敗例、注意点、改善履歴]
【承認】[承認者/承認日/適用範囲]
【改訂履歴】[版、変更点、変更理由]
```

**運用ルール（最低限）**
- 命名規則: 目的が分かる名称＋用途カテゴリ（例: `meeting/minutes-v1`）
- 更新責任: オーナーを明確にし、定期レビュー（例: 四半期）を設定する
- 廃止ルール: 使われない/危険が高い/前提が古いプロンプトは無効化し、理由を残す

### A.5.3 演習の採点基準（ルーブリック）テンプレート

演習を研修・教材として運用する場合は、採点者のばらつきを減らすためにルーブリックを用意します。CARE（第6章）をベースにしつつ、章ごとの到達目標に合わせて配点を調整できる形にします。

**ルーブリック（雛形）**
```text
【採点対象】（例: 第10章 演習10-1 / 提案書の骨子）
【満点】100点

1) Correctness（正確性）: 0〜5点 × 配点（例: 4）= 0〜20点
   - 事実と推測が区別されている
   - 根拠のない数値や断定がない

2) Appropriateness（適切性）: 0〜5点 × 配点（例: 4）= 0〜20点
   - 読み手・目的・文体が適切
   - 機密配慮（抽象化/匿名化）ができている

3) Relevance（関連性）: 0〜5点 × 配点（例: 4）= 0〜20点
   - 問いに正面から答えている
   - 不要な脱線がない

4) Effectiveness（効果性）: 0〜5点 × 配点（例: 4）= 0〜20点
   - 次アクションに繋がる
   - 意思決定に必要な材料が揃っている

5) 章固有の観点（0〜20点）
   - 第10章（文書）: 結論先出し/構造（見出し）/根拠と結論の対応
   - 第12章（会議）: 決定事項/ToDo（担当・期限）/未決の分離
   - 第13章（交渉）: 目的/譲歩条件/想定反論と回答方針
```

**運用のポイント**
- 採点前に「前提（想定読者・用途・制約）」を揃える
- 不確かな箇所は減点ではなく「要確認」としてフィードバックに残す
